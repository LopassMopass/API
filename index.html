Blog API Documentation

This API allows users to create, read, update, and delete blog posts with user-specific permissions. Each user can only edit or delete their own posts, and they can control visibility of their posts. An admin user has full access to all posts.

Base URL

http://localhost:3000/api

Authentication

This API uses Basic Authentication. Users must provide their username and password for all protected routes.

Endpoints

GET /api/about

Description: Fetch all possible endpoints in Blog-API.

Example Request:

GET http://localhost:3000/api/about

GET /api/blogs

Description: Fetch all blog posts. Accessible to all users.

Example Request:

GET http://localhost:3000/api/blogs

GET /api/blogs/:blogId

Description: Fetch a single blog post by ID. Accessible if user has permission.

Example Request:

GET http://localhost:3000/api/blogs/1

POST /api/blogs

Description: Create a new blog post. Accessible to authenticated users.

Request Body in JSON format like this:

{"text": "My first blog post", "date_of_creation": "2024-11-03"}

Example Request:

POST http://localhost:3000/api/blogs
                Content-Type: application/json
            {
                "text": "My first blog post",
                "date_of_creation": "2024-11-03"
            }

Error Codes

  • 401 Unauthorized: User is not authenticated.
  • 403 Forbidden: User does not have permission for the requested operation.
  • 404 Not Found: The specified blog post or resource was not found.
  • 500 Internal Server Error: An error occurred on the server.
